* Trạng thái của file:
	- uncheked (khi tạo mới 1 file mà chưa được git quản lí)
	- stage (sau khi git add <tên file> or git add . or git add --all) -- chuẩn bị commit
	- unmodifile (khi file đã được quản lí và đã commit rồi)
	- modifile (file có thay đổi sau khi add thì về stage)
* Câu lệnh chú ý trong git
	- git config --gobal user.name (user.email)
	- git cd "" (đi vào thư mục)
	- git init (tạo git)
	- git status
	- git add (add . , add --all, add <tên file>)
	- git restore --staged <tên file> (chuyển từ stage về unstage)
	- git restore <tên file> (xóa bỏ sự thay đổi của file)
	- git diff (Xem thay đổi của file)
	- git checkout -b <tên nhánh> (tạo nhánh phụ)
	- git checkout <tên nhánh>
	- git branch (xem ở nhánh nào)
	- git merge <tên nhánh phụ> khi bạn ở master mới ghép được
	- git checkout -d <tên nhánh> (xóa bỏ nhánh)
	- git commit -m "message"
	- git show <code>
	- git log
	- gitk (show giao diện)
	- git reset --sort <code> (loại bỏ commit đến vị trí code , file ảnh hưởng về stage -- sẵn sàng commit)
	- git reset --mixed <code> (loại bỏ commit đến vị trí code , file ảnh hưởng về unchecked -- sẵn sàng add)
	- git reset --hard <code> (loại bỏ hết)
	- git revert <code> xóa bỏ thay đổi của commit <code> và tạo ra 1 commit mới (+ thành -)
	- git push origin <branch> (push lên git với nhánh branch)
	- git pull (update lại phía local)
